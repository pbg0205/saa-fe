# 작업 내용

- 주요 작업 내용을 정리.
- 작업 내용은 명사형으로 정의.
- 무엇을 개선/추가/수정할 것인지 명확하게 작성.

>예시)
><br>
>
>- 헤더 컴포넌트의 디자인 개선
>- 반응형 레이아웃 적용
>- 네비게이션 스타일링 개선

# 작업 순서

- 작업 내용과 순서를 정의.
- 작업 간 의존성을 고려하여 순서 결정.

>예시)
><br>
>
>1. 현재 코드 분석 및 문제점 파악
>2. 디자인 시스템 적용
>3. 반응형 디자인 구현
>4. 테스트 및 검증

# TO-DO-LIST

- 작업 순서에 따른 세부 TO-DO-LIST 작성.
- 체크박스를 사용하여 진행 상태 추적.
  - `[ ]`: 미완료
  - `[x]`: 완료

>예시)
><br>
>
>1. 현재 코드 분석 및 문제점 파악
>   - [x] 현재 Header 컴포넌트 구조 분석
>   - [x] 개선이 필요한 부분 목록화
>   - [ ] 디자인 시스템 문서 검토
>2. 디자인 시스템 적용
>   - [ ] 색상 시스템 적용
>   - [ ] 타이포그래피 스타일 적용
>3. 반응형 디자인 구현
>4. 테스트 및 검증

# 참고 파일

- 작업과 관련된 파일 경로 및 문서 링크 작성.

>예시)
><br>
>
>- `src/components/Header.tsx`
>- `documents/design/DESIGN_SYSTEM.md`
>- `documents/plan/PROJECT_PLAN.md`

# 작업 완료 내역

작업이 완료되면 아래 형식으로 기록.

## N차 작업 (YYYY-MM-DD)

### 문제 상황
- 발견된 문제나 개선이 필요한 상황 설명

### 원인 분석
1. 원인 1
2. 원인 2

### 개선 작업

#### 세부 작업 제목
- 구체적인 개선 내용
- 적용한 기술이나 방법

### 결과
- ✅ 개선 완료 항목 1
- ✅ 개선 완료 항목 2
